## 이미지 (`blog/images/`)
스토어 스크린샷 재활용 + 블로그 전용 2장(howto, compare). 전부 개인정보 없는 샘플 데이터.

## 발행 팁
- **한 번에 다 올리지 말 것.** 주 1~2편씩 나눠 올리면 자연스럽고 검색엔진에도 유리.
- 추천 순서: 1(추출법) → 3(경비정산) → 6(오빠두) → 2 → 5 → 4. 검색량 큰 것부터.
- 제목은 파일의 `<h1>`을 그대로. 태그는 글 맨 끝 해시태그 참고.
- 발행 후 각 글 댓글/방명록 문의는 초안 도와줄 수 있음.
- 이미지는 캡션까지 같이 넣으면 체류시간·가독성에 좋음.

## 톤 원칙 (다음 글도 이 톤 유지)
- 1인칭, 반말 회고체. "나는 쿠팡을 꽤 많이 쓴다… 그래서 직접 만들었다."
- 광고 문구 금지("지금 바로!", "최고의!" X). 실제 겪은 불편 → 해결 흐름.
- "직접 만든 확장"이라는 솔직한 프레임 유지(개발 블로그라 신뢰도↑).
- 개인정보(로컬 저장·읽기 전용)는 매 글 자연스럽게 한 번 언급.
